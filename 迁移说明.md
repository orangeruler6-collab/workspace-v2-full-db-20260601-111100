# Workspace V2 迁移包说明

打包时间：2026-06-01

## 已包含

- 前端/后端源码：`src`、`server`、`scripts`、`tools`
- 构建产物：`dist`
- 主项目配置：`package.json`、`package-lock.json`、`vite.config.js`
- 数据库：`data/*.db`、`server/data/*.db`
- 上传和生成素材：`public/uploads`
- 静态资源：`public/assets`
- 子应用源码：`apps/account-style-library`
- 文档和启动脚本：`README.md`、`START_HERE_启动项目.md`、`一键启动.bat`、`一键停止.bat`

## 已排除

- 密钥文件：`.env`、`.env.local`
- 依赖目录：`node_modules`
- Python 虚拟环境：`.venv`
- 运行缓存：`.runtime`
- 临时目录和旧备份：`temp`、`tmp`
- 构建/开发缓存：`.next`、`.cache`、`.turbo`
- 日志和进程文件：`*.log`、`*.pid`

## 迁移后需要手动补

1. 把原机器的 `.env` 按需复制到新机器项目根目录，或参考 `.env.example` 重新填写。
2. 如果使用 `apps/account-style-library`，把它自己的 `.env.local` 按需补到 `apps/account-style-library/.env.local`。
3. 发布模块涉及 OpenCLI/Chrome profile，需要在新机器重新安装 OpenCLI、创建对应 profile，并登录账号。
4. 如果新机器需要视频转写/音频处理，确认 ffmpeg 可用；原 `.runtime` 没打包，避免把浏览器缓存和本机运行垃圾带过去。

## 常用启动步骤

```powershell
npm install
npm run build
node server/index.cjs
```

如需开发模式，再按项目现有脚本启动前端服务。
