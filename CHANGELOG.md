# CHANGELOG - 乌萨奇工作平台 v2

格式规范：
- `### 新增` — 新功能上线
- `### 修改` — 既有功能改动
- `### 修复` — bug修复
- `### 删除` — 废弃功能移除

每次重要改动同步更新此文件。

---

## 2026-05-25

### 新增
- 数据维护板块新增按内容组隔离的历史记录：自动识别当前用户所属组，同一标的下可保留一期、二期等分期维护记录。
- 数据维护新增抖音 CPM 预设能力：支持目标 CPM、分期比例、组内默认互动比例，并按投放区间向上取整。
- 数据维护新增组内配置：各内容组可单独调整默认预设比例和平台单价，互不混用。

### 修改
- 更新公告切换为本次数据维护说明，提醒各组每期按实际情况上下浮动调整，避免数据过于平整。

## 2026-04-29

### 新增
- 顶栏新增 UI 风格切换：支持“紫夜”和“银白”两套全局配色，并自动记住本地选择。
- 新增后端认证、管理员账号、模块权限控制与操作日志模块。
- 新增权限管理后台：支持创建账号、启停账号、调整角色与模块权限、重置密码。
- 新增操作日志后台：支持按用户、模块、动作和日期筛选登录及关键写操作。
- 登录页新增自助注册入口：仅允许中文真名注册，需填写邀请码，注册后默认拥有全部业务模块权限。
- B站工具接入 `tools/bilibili-cli` Git Submodule，后端优先用 bilibili-cli 获取字幕，无字幕时下载音频并走硅基流动转写。

### 修改
- 第二套 UI 改为偏 Apple 的白色玻璃质感，并加入轻微 hover、press、卡片浮起等微交互。
- 每日热点、创意看板、文案工作流画布接入主题变量，银白主题下同步切换为浅色面板和画布。
- 登录方式从前端本地统一密码改为后端会话 token，并由后端统一校验 API 权限。
- 创意看板和素材库的操作者来源改为当前认证用户，减少本地用户名被篡改的风险。
- 文案工具移除分区说明块，四张工具卡统一为对齐网格排版，桌面双列、窄屏单列。
- 后端检查脚本新增 B站工具检查，并在缺少 `.venv`/`bili` 命令时给出初始化命令。
- B站文案转写卡片对齐抖音文案转写工作流，新增转写策略、进度条、来源信息、错误面板和统一结果操作。
- 抖音/B站转写结果的“复制文案”改为“生成文案”，直接把转写稿带入文案生成的素材栏；转写结果区移除“写入飞书”。
- 抖音/B站转写结果的“AI纠错排版”改为本地“清洗分段”，避免整段文案走模型导致等待过长。

### 修复
- 修复开发模式只启动前端导致 Vite 代理 `/api` 到 5555 端口时出现 `ECONNREFUSED`；`npm run dev` 现在会同时启动 API 与前端。
- 补充内联 favicon，避免浏览器请求 `/favicon.ico` 产生 404 噪声。
- B站转写接口不再吞掉错误为空结果，改为向前端返回明确失败原因。
